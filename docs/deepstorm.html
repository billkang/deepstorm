背景与问题

AI 辅助开发的现状与局限

代码生成速度的提升，并不直接等同于需求理解准确度、系统设计一致性和工程可维护性的提升。

🔄

信息传递衰减与失真

从产品需求到技术实现的多层翻译过程中，原始意图容易偏离。信息在角色间传递时持续衰减，团队需要大量反复沟通来对齐理解。

🧩

上下文碎片化

AI 缺乏持续稳定的工程上下文，每个会话独立理解任务，无法感知上游决策、业务背景和架构约束，生成结果容易偏离实际意图。

🔗

需求·规范·测试 脱节

需求、规范、测试、实现之间缺少显性关联，变更影响难以快速追踪，规约与历史决策无法沉淀为长期工程资产。

三个递进目标

1
效率提升 利用 AI 辅助代码生成、测试生成、规范草拟，减少重复性手工劳动，加快从需求到交付的流转速度。
2
上下文连续性 通过结构化规格文档将需求、接口规范、测试用例和实现代码串联为一条显性链路，降低因信息断层导致的理解偏差和返工。
3
规约资产化积累 每次需求澄清、设计决策和验收标准制定都以结构化形态沉淀，构成可追溯的历史记录，使变更影响评估有据可依。
当前 AI 辅助开发提升了单点的执行速度，却未能解决跨环节的上下文断层。
这正是 SDD 能够弥补的部分——以结构化规格文档作为各环节的共同输入，使需求、规范、验证之间保持显性关联。
核心思路

以 Spec 为轴的 AI 协同工程

利用 AI 提升各环节执行效率，同时以结构化 Spec 将需求、规范、验证、实现串联，使 AI 和人在一致的上下文中协作。

01

需求质量前置

利用 AI 辅助的结构化追问与校验，在需求进入开发前澄清边界条件、异常路径与验收标准，将缺陷拦截在产品阶段。

02

上下文连续性

通过统一的 feature_id 将需求、接口规范、测试用例、实现代码显性关联，使 AI 在任何步骤都能获取上游结构化信息。

03

人机边界清晰

人负责决策（目标、约束、策略），AI 负责生成与执行（结构化、草稿、代码）。所有 AI 产出均需人工确认，不绕过判断上线。

三层 Spec 体系

结构化的工程规约链路

三层 Spec 之间为单向驱动关系：上层产出作为下层输入，形成信息密度高、传递损耗低的工程链路。

第一层 · Business Spec

业务规范

业务目标、用户目标、验收标准、Story 列表。由产品侧维护，需求驱动更新，写完归档为主，部分修改可追加。

消费者：产品 消费者：开发
工具：BMAD
→ 驱动
第二层 · Technical Spec

技术规范

接口规范、数据模型、业务规则、开发约束。由开发侧维护，迭代驱动持续演化，反映系统当前真实状态。

消费者：开发 消费者：AI 代码生成
工具：OpenSpec
→ 驱动
第三层 · Verification Spec

验证规范

测试用例、断言、预期行为（正常/异常/边界）。由测试侧维护，随功能和缺陷修复持续补充。

消费者：测试 消费者：AI 测试生成
工具：Superpowers
feature_id: AUTH-LOGIN-WECOM
● Business Spec → epics/epic-1.md 中的对应 Story
● Technical Spec → POST /auth/wecom、数据字段定义
● Verification → invalid_qr / expired_qr / unauthorized_org 测试场景
● 实现代码 → auth/wecom.ts
模式对比

传统模式困于人工翻译损耗，当前 AI 模式快在单点却未解决断裂，DeepStorm 用结构化规约让 AI 在连续上下文中系统工作。

工具套件

DeepStorm 生态体系

四个子套件分别面向不同角色，命名源自珊瑚礁生态系统，隐喻个体协作构建复杂系统的工程理念。

🌊

Tide

潮汐 · Desktop Extension

需求流人的源头。负责需求结构化、PRD 管理、Jira 同步，为整个链路提供驱动力。

生态隐喻：潮汐，养分与势能的周期性输入
🪸

Reef

礁群 · Claude Code Plugin

代码的持续构建与结构沉淀。负责规范生成、代码实现、架构约束检查。

海洋隐喻：珊瑚礁群由无数珊瑚虫经年构建
🌊

Sweep

扫掠 · Claude Code Plugin

持续感知代码质量。负责测试生成、覆盖率分析、CI 结果解读，异常的第一道探测器。

生态隐喻：与珊瑚共生，对水质极其敏感
🏝️

Atoll

环礁 · Desktop Extension

系统最终运行的稳定承载层。负责部署辅助、监控查询、故障排查和环境检查。

生态隐喻：珊瑚礁发育的最高阶段
完整工程链路

从产品意图到自动化测试的全流程

链路的核心运行方式：产品输出高质量 PRD 后创建 Jira 任务，开发人员在新迭代中自行拉取任务，利用 AI 工具逐步完成 Spec 生成、代码实现和测试。除 CI 外，多数环节由人工按需触发。

价值与路径

预期价值与实施路径

AI 加速 + 结构化上下文 → 将信息不对齐和重复手工劳动带来的损耗控制在较低水平，让团队聚焦高价值决策和创造性工作。

✅

Tide — 产品需求结构化

使用 BMAD 在 AI 辅助下将自然语言意图转化为结构化需求文档，产出 PRD + Story + 验收标准，并同步至 Jira 任务管理，为整条链路提供驱动力。已实现。

✅

Reef — 规范生成与代码实现

基于 Business Spec 自动生成 Technical Spec（接口定义、数据模型、业务规则），建立 feature_id 关联规则和 Spec 文件组织约定，并通过 hooks 在 Claude Code 中执行自动格式化、文件保护和运行测试。已实现。

✅

Sweep — 测试生成与覆盖率分析

从验收标准半自动生成测试用例，覆盖正常路径、异常路径和边界条件，并将测试场景转化为可执行的单元测试和集成测试。结合 CI 管线自动触发测试执行，结果快速反馈。已实现。

○

Atoll — 部署辅助、监控与运维排查（暂未实现）

部署辅助、监控查询、故障排查和环境检查。作为系统的稳定承载层，待后续迭代接入。

这套实践旨在利用 AI 提升开发效率的同时，以结构化规格文档将产品、开发、测试的上下文串联。
它是对 SDD、TDD 等既有范式的工程化应用，存在磨合成本和暂时无法解决的问题。
但相比于团队各自为战、文档与代码脱节的现状，朝这个方向推进，
我们认为会更接近高质量的持续交付。
