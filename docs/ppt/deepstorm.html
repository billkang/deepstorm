背景与问题

AI 辅助开发的三大核心问题

AI 编码速度已大幅提升，但需求传递、上下文衔接、规范落地的协作方式没有跟上——真正的瓶颈不在代码生成，而在结构化信息的传递效率。

🔄

需求传递中的信息衰减

PM 在 PRD 描述的功能，开发理解时已偏离 30-50%。口头讨论无结构化记录，团队反复对齐却总有遗漏。最终交付与预期不符，返工成为常态。

🧩

AI 协作的上下文断裂

每次 AI 会话从零理解，无法感知架构约束和上游决策。开发者需要反复为 AI 补充项目背景。Vibe Coding 模式下，工程知识随会话关闭而消逝，无法沉淀复用。

🔗

需求·设计·实现·测试脱节

需求变更无法追溯影响范围，测试用例靠人工维护。code-style 依赖口头约定，新人加入后迅速走样。工程最佳实践随人员流动而流失，团队经验留不住。

核心思路

02

Reef · 能力体系

Skills · Agents · Hooks 三层能力矩阵

每层解决不同维度的问题：Skills 解决"用户如何触发能力"，Agents 解决"多步骤工作流自动编排"，Hooks 解决"开发流程中自动化门禁"。

🪸 Skills
用户主动触发 · 覆盖开发全步骤
/reef-start
启动唯一入口，自动路由 Issue 或自由讨论，拉取 PRD/Figma/上下文，触发完整 SDD 流程
/reef-gen-frontend / reef-gen-backend
生成按 code-style 规范自动生成组件/服务/路由骨架，代码与团队约定自洽
/reef-testcase
测试从 Issue + PRD 提取四维测试场景：正常流程 / 边界条件 / 异常场景 / 验收标准
/reef-review
审查多子 Agent 并行审查（正确性/性能/安全/可维护性），逐维评分汇总
/reef-harden
加固实现后审视设计，含"反向 grill"——追问三个月后可能在哪五处后悔
/reef-scope
范围AI 语义分析 diff，检测多领域变更并阻断，给出拆分建议。支持 pre-commit + CI
/reef-commit / reef-pr
提交智能提交信息（继承仓库风格）→ 自动 PR（含变更背景 + Issue 关联），AI 草拟人确认
⚙️ Agents
后台自动编排 · 多步骤工作流
OpenSpec Agent
生成执行 proposal → design → tasks 三阶段 SDD，OpenSpec 格式约束保证文档质量
grill-me Agent
审查逐阶段 SDD 多轮追问，发现遗漏和矛盾才进入下一阶段，确保设计严谨性
spec-hardener Agent
加固四道筛：完整性 / 一致性 / 异常路径 / 反向 grill，将边缘问题消灭在设计阶段
Review Agents
审查每个维度独立子 Agent 并行执行：正确性 / 性能 / 安全 / 可维护性，综合评分
Pilot Agent
自动Harness 自动化编排 —— 监听 OpenSpec 变更，跨套件自动触发工程全链路流转
🔗 Hooks
环境嵌入 · 自动化流程门禁
pre-Agent 上下文注入
上下文Agent 执行前自动注入当前任务的 PRD / Spec / Figma 上下文
post-Agent 产出校验
验证Agent 生成结果的格式合规性、规范符合度自动检查
pre-commit scope 检查
门禁提交前自动执行 reef-scope 语义分析，阻断多领域混合提交，确保 commit 粒度纯洁
commit-msg 规范校验
门禁自动校验提交信息格式和风格一致性，继承仓库历史习惯
pre-push 质量门禁
门禁推送前自动触发测试执行 + lint 检查，阻断低质量代码流入远端
CI 分支范围验证
门禁在 GitHub Actions / GitLab CI 中运行，PR 级别检测多领域变更

模式对比

Vibe Coding vs DeepStorm 结构化工程

核心差异不在于代码生成速度，而在于是否将工程知识沉淀为可持续积累、可系统化管理的资产。

维度 Vibe Coding DeepStorm 结构化工程
工作模式 即兴提示，AI 自由发挥 Spec 驱动的结构化链路，按规约执行
上下文连续性 每次会话独立，历史即消逝 feature_id 贯穿，Spec 持续传递上下游
文档资产积累 ❌ 无，会话即消逝 ✅ OpenSpec 文档持续沉淀，可查阅可继承
PRD 管理 ❌ 无 ✅ Tide 全生命周期管理（讨论→发布→变更→归档）
代码质量 依赖训练数据，风格随机 ✅ code-style 规范约束 + 多维 Agent 审查
项目质量保障 无系统机制，生成即交付 ✅ 四维质量门禁：用例→范围验证→审查→加固
可追溯性 ❌ 无，需人工回忆 ✅ Issue → Spec → Code → PR 全链路关联
自主可控 平台锁定，无法定制 ✅ 各环节可替换（MCP/语言/规范），公司规范可注入
规约自定义 ❌ 不可 ✅ code-style 维度可组合、template 可扩展、工作流按需调整
