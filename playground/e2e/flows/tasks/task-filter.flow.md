## Flow: F01 - 按状态筛选任务

### 前置条件
- 应用已启动（`cd ../app && npm start`）
- 用户 `filtertest_user` 已注册并登录（通过 L01 流程）
- 存在多个任务，包含至少 1 个待办和 1 个已完成（可通过 T01/T04 创建）

### 执行步骤
1. 导航到仪表盘 `http://localhost:3000/dashboard.html`
   ✅ 验证点：任务列表显示所有任务，统计信息正确
2. 在状态筛选下拉框中选择"已完成"
   ✅ 验证点：任务列表仅显示已完成任务
   ✅ 验证点：列表中的待办任务被隐藏
   ✅ 验证点：统计信息更新为仅显示已完成的数量
3. 在状态筛选下拉框中选择"待办"
   ✅ 验证点：任务列表仅显示待办任务
   ✅ 验证点：列表中的已完成任务被隐藏
4. 在状态筛选下拉框中选择"全部状态"
   ✅ 验证点：任务列表再次显示所有任务

### 环境要求
- 目标环境：test
- 所需账号：filtertest_user（已登录）

## Flow: F02 - 关键字搜索任务

### 前置条件
- 应用已启动
- 用户 `filtertest_user` 已登录
- 存在任务"购买办公用品"和"提交周报"（通过 T01 创建）

### 执行步骤
1. 导航到仪表盘 `http://localhost:3000/dashboard.html`
   ✅ 验证点：任务列表显示所有任务
2. 在搜索框中输入关键字"办公"
   ✅ 验证点：任务列表仅显示包含"办公"的任务（"购买办公用品"）
   ✅ 验证点：不相关的任务（"提交周报"）被隐藏
3. 点击"Enter"触发搜索
   ✅ 验证点：搜索结果保持不变
4. 清空搜索框
   ✅ 验证点：任务列表恢复显示所有任务

### 环境要求
- 目标环境：test
- 所需账号：filtertest_user（已登录）

## Flow: F03 - 搜索不存在内容

### 前置条件
- 应用已启动
- 用户 `filtertest_user` 已登录
- 存在至少一个任务

### 执行步骤
1. 导航到仪表盘 `http://localhost:3000/dashboard.html`
2. 在搜索框中输入不存在的关键字 `xyz_nonexistent_abc`
   ✅ 验证点：任务列表显示"暂无任务"空状态提示
   ✅ 验证点：统计信息显示为 0（筛选条件下无匹配）
3. 清空搜索框
   ✅ 验证点：任务列表恢复显示所有任务
   ✅ 验证点：统计信息恢复正常

### 环境要求
- 目标环境：test
- 所需账号：filtertest_user（已登录）
