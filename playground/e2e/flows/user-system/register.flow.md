## Flow: R01 - 正常注册新用户

### 前置条件
- 应用已启动（`cd ../app && npm start`）
- 测试用户 `testuser_001` 尚未注册

### 执行步骤
1. 导航到注册页面 `http://localhost:3000/register.html`
   ✅ 验证点：页面标题包含"注册"，注册表单可见（含用户名和密码输入框）
2. 在用户名输入框中输入 `testuser_001`
   ✅ 验证点：输入框显示输入内容
3. 在密码输入框中输入 `test1234`
   ✅ 验证点：输入框显示输入内容
4. 点击"注册"按钮
   ✅ 验证点：页面显示"注册成功"提示，约 1.5 秒后跳转到登录页
5. 确认跳转到登录页
   ✅ 验证点：当前 URL 为 `http://localhost:3000/`，登录表单可见

### 环境要求
- 目标环境：test
- 所需账号：无

---

## Flow: R02 - 重复用户名注册

### 前置条件
- 应用已启动
- 用户 `testuser_002` 已注册（通过 R01 流程或直接注册）

### 执行步骤
1. 导航到注册页面 `http://localhost:3000/register.html`
   ✅ 验证点：注册表单可见
2. 使用 R01 中已注册的用户名 `testuser_002` 再次填写表单
3. 输入密码 `test1234`
4. 点击"注册"按钮
   ✅ 验证点：页面显示错误提示"用户名已存在"，页面未跳转

### 环境要求
- 目标环境：test

---

## Flow: R03 - 密码太短

### 前置条件
- 应用已启动
- 为新用户准备不同的用户名字段

### 执行步骤
1. 导航到注册页面 `http://localhost:3000/register.html`
   ✅ 验证点：注册表单可见
2. 输入用户名 `testuser_shortpw`
3. 输入密码 `12`（仅 2 位，少于最低 4 位要求）
4. 点击"注册"按钮
   ✅ 验证点：页面显示错误提示"密码至少 4 位"，页面未跳转

### 环境要求
- 目标环境：test
